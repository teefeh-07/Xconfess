# Contract Deployment Records

This directory stores generated metadata for xConfess Soroban contract builds
and network deployments.

## Files

- `contract-wasm-manifest.json` is generated by
  `./scripts/contracts-release.sh build`. It records release WASM paths,
  crate versions, byte sizes, and SHA-256 hashes before deployment.
- `<network>.json` is generated by
  `./scripts/contracts-release.sh deploy --network <network> --source <key>`.
  Use the Stellar CLI network alias as the filename, for example
  `deployments/testnet.json` or `deployments/public.json`.
- `sample.redacted.json` is an example only. It documents the expected shape
  without real contract IDs or deployer information.

## Naming Convention

Deployment metadata must use:

```text
deployments/<network>.json
```

`<network>` must match the network argument passed to the release script. If the
team uses `public` as the Stellar CLI alias for mainnet, commit
`deployments/public.json`. If the team explicitly configures a `mainnet` alias,
commit `deployments/mainnet.json` and mention the alias in the release notes.

## What To Commit

Commit:

- reviewed `deployments/<network>.json` files for shared environments
- `deployments/contract-wasm-manifest.json` when publishing a release artifact
  snapshot for review
- documentation examples such as `sample.redacted.json`

Do not commit:

- private keys, secret keys, seed phrases, or local `.env` files
- one-off local deployment records from personal experiments
- generated build directories such as `xconfess-contracts/target/`
- unreviewed metadata where hash changes cannot be traced to source changes

Before committing a deployment file, verify the hashes with the checksum
workflow in `docs/contract-release-and-upgrade-runbook.md`.
